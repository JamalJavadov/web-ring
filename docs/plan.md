# Goal Description
Implement an automatic product generator script for the frontend-only gothic accessories store that scans specific image folders and outputs a typed product list array.

## Objective
Automatically scan `public/products/uzukler` and `public/products/sepler` directories and generate `src/data/products.generated.ts` containing a TypeScript array of products based on fixed pricing rules and filename conventions.

## Scope
- **In Scope:**
  - Create a Node.js script (`scripts/generate-products.mjs`) using only Node.js built-ins (`fs/promises`, `path`).
  - Read images from specific categorized public folders.
  - Apply fixed price rules based on folder (`uzukler` -> 13 AZN, `sepler` -> 15 AZN).
  - Parse filenames to derive product slug and group multiple images (e.g., `-1`, `-2`).
  - Auto-generate title-cased names from slugs and assign tags.
  - Sort products by file creation time descending.
  - Provide empty arrays if folders and images are missing.
  - Write standard exported TypeScript type and array to `src/data/products.generated.ts`.
  - Add standard npm scripts (`gen:products`, `predev`, `prebuild`).
  - Update `README.md` with "Adding Products" instructions.
  - Replace imports across the application that currently use `src/data/products.ts` with the injected `src/data/products.generated.ts`.
- **Out of Scope:**
  - Image manipulation or resizing.
  - Backend API integration or actual database connections.
  - Modifying non-product components.

## Assumptions
- Node.js is installed to execute the script.
- The default material is always `['metal']`.
- Images are named correctly following the slug or `slug-N` format for grouping and `.webp`, `.png`, `.jpg` format.

## Constraints
- **Minimal Diffs:** Change only what is necessary in application files to switch imports to the new generated file.
- **No Comments/TODOs:** Target files will have no inline TODOs or comments (other than the required auto-gen header).
- **No wildcard imports:** Strict TS imports only.
- **Perfect Naming:** Variables within the script must be clear. Script must be deterministic across OS (Win/Mac/Linux).

## Proposed Changes

### Configuration
#### [MODIFY] `package.json`
- Add `"gen:products": "node scripts/generate-products.mjs"`
- Add `"predev": "npm run gen:products"`
- Add `"prebuild": "npm run gen:products"`

#### [MODIFY] `README.md`
- Append an "Adding Products" section documenting the required folder structure, naming conventions, and image grouping rules.

### Script
#### [NEW] `scripts/generate-products.mjs`
- Contains logic to perform `fs.readdir` recursively on the target folders.
- Implements grouping logic (by regex removing `-N` suffix).
- Builds product attributes from slug.
- Sorts by file modification time (simulating creation time or using `fs.stat`).
- Outputs `src/data/products.generated.ts`.

### Application Data Source
#### [DELETE] `src/data/products.ts`
- Remove the static list.
#### [NEW] `src/data/products.generated.ts`
- (Generated by script) contains `export type Product` and `export const products: Product[]`.

### Application Usage
#### [MODIFY] relevant files (e.g., `src/pages/Shop.tsx`, `src/pages/Home.tsx`, `src/pages/ProductDetail.tsx`, etc.)
- Update imports from `@/data/products` to `@/data/products.generated`.

## Step-by-Step Implementation
1. Add `scripts/generate-products.mjs` script logic.
2. Update `package.json` with the new npm scripts.
3. Run `npm run gen:products` to create `src/data/products.generated.ts`.
4. Replace all imports pointing to the old data file to use the new generated file.
5. Delete `src/data/products.ts`.
6. Append the required documentation to the `README.md`.

## Risks + Mitigations
- **Cross-Platform Determinism:**
  - *Risk:* Order of products or generated file paths might mismatch on Windows (`\` vs `/`).
  - *Mitigation:* Explicitly use `path.posix` for generating URL paths inside the script, and enforce consistent sorting (e.g., by stats + slug fallback).

## Acceptance Criteria
- Running `npm run gen:products` successfully generates `.ts` without heavy dependencies.
- Prices correctly reflect 13 AZN for rings and 15 AZN for necklaces.
- Images grouped properly by slug base name.
- Building the app via `npm run build` implicitly calls the script.
- The app successfully compiles and fetches data from `products.generated.ts`.
